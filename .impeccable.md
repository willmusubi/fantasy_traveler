# Design Context — Fantasy Traveler

## Target audience
A self-improver (the user themself, and people like them) who lives in a todo/calendar/journal
but finds plain productivity tools sterile. They want progress to feel **rewarding, warm, and
game-like**, and want an AI companion that "gets" them. Personal/individual use.

## Use cases
- Capture and complete real-life Todos; journal; plan on a calendar.
- See those real actions move a Final-Fantasy-style pixel RPG (damage a 拖延心魔, gain XP).
- Build a Fire-Emblem-style 好感度 bond and chat with AI companions (the 无期迷途 / 来生 sisters).

## Brand personality / tone
Warm, nostalgic, encouraging JRPG companion. Never sterile, never harsh. It should feel like
sitting down to a cozy Super Famicom RPG that happens to run on your real life.

## Committed aesthetic direction (chosen 2026-05-29)
**Classic 16-bit JRPG window (Final Fantasy / SNES).** The whole UI is framed like an FF command
menu:
- **Signature element:** deep royal-blue gradient windows with a rounded **cream/gold double
  border** (dark outline → cream frame → blue fill with a top highlight + faint inner gold line).
  Recognizably "SNES Final Fantasy menu," not a generic card.
- **Color:** night-indigo background with a faint starfield; royal-blue windows; **gold** for
  titles/selection/level; cream-white ink; periwinkle (blue-tinted, never gray) muted text. HP
  red, affinity pink, all harmonized to the blue/gold world. No neon, no purple→blue gradients.
- **Type:** **Zpix** pixel font (CJK + Latin) for the authentic bitmap feel, with pixel
  1px drop-shadows on headings; graceful fallback to a clean CJK sans if the font fails.
- **Cursor:** a gold **▸** selection pointer appears on hover/active rows (the FF menu cursor).
- **Motion:** windows "open" with a quick scale/fade; buttons press down (translateY) like
  physical SNES menu commands; restrained, no bounce.
- **Hero / signature:** a full-width **FF battle stage** at the top — your party (player +
  companions, as sprites) stands on a dusk scene (gradient sky, moon, stars, ground plane)
  facing the 心魔, with a boss HP bar below. Completing real tasks visibly attacks it. This,
  plus the command-window frame + gold ▸ cursor + starfield, makes it unmistakably a JRPG, not
  an AI-generated dashboard. Layout has a clear hero (battle) → quest log (todos) → companion
  hierarchy rather than a uniform grid of identical windows.
- **Motion:** windows open with a quick scale/fade; HP/affinity bars have a shine sweep;
  sprites idle-bob; the monster flashes when low; the selection ▸ blinks. All respect
  prefers-reduced-motion.

Two-zone discipline still applies: dense reading (chat/journal) stays legible (larger pixel
text, generous line-height); chrome/labels use the tighter pixel sizing.

**Character portraits vs UI chrome (clarified 2026-06-06):** the UI **chrome** stays 16-bit SNES
(blue/gold windows, Zpix font, ▸ cursor). **Character art** is **HD anime splash**, matching the
来生 三姐妹 reference set in `docs/avator-pics/`, generated via the `character-art` skill — which ALSO
pixelizes a 16-color variant for the compact in-game bust. Net: HD anime portraits inside a pixel
frame. (Resolves the earlier "splash vs pixel" ambiguity between this file and `portrait-prompts.md`.)

## Worldview + storyline surfaces (added 2026-05-29)
The worldview arc's new screens stay in the same FF idiom: **world select** and **quest board**
are command-window lists with the gold ▸ cursor; the **battle stage becomes narrated** (quest
title/lore + per-encounter intro/victory text in an FF dialogue box, "第 N/总" progress, the enemy
shows its generated name/theme); **recruit** is a celebratory FF window with the new character's
portrait; **party** (≤6) and **equipment** panels use the same framed rows + stat readouts with
synergy badges. New characters from other worlds must still pass through the same Style Bible
(Appendix A) so the roster reads as one universe.
